2) ARCHITECTURE DECISIONS LOCKED THIS SESSION
============================================================

A) ODDS ↔ VALUE FIREWALL (hard rule)
   Three SEPARATE, non-cross-contaminating concerns:
     1. Value panel value = OUR statistical estimate (form/history/standings,
        Poisson). Computed ONCE → FROZEN, stable, never changes. NOTHING to do
        with odds.
     2. Odds (two UI panels) = real scraped bookmaker odds, shown ONLY for drift
        (open→current→delta). Display layer. May at most signal a match EXISTS —
        never an input to value.
     3. Per-match DETAILS assessment = what the AI writes in each match's details,
        for ALL matches. A DISTINCT output from (1) — NOT the same number as the
        value-panel value. Also odds-independent.
   Value panel (1) and details assessment (3) are two different things — do not
   conflate them.

B) DATA CLASSIFICATION
   - deploy-snapshots/<day>/fixtures.json  = PREFERRED display source (ESPN
       canonical status/scores), but a regenerable deploy artifact — NOT truth.
   - odds.json / fixtures-all.json / multi-odds = SUPPLEMENTS (rows/existence +
       odds drift). Never a status/result authority. Never feed value.
   - data/fixtures.json (json-db) = regenerable OPERATIONAL BRIDGE/CACHE, NOT a
       statistical truth store. Frozen openings mirror to data/assessments/<day>.json.
       => run-odds-opening.js "canonicalWrites: 0" wording is acceptable ONLY under
          this classification; make comments/guardrails say so explicitly.
   - Statistical TRUTH = league-memory (results / standings / history). Deploy
       snapshots + bridge must never overwrite/contaminate it.

C) DAILY-RUN ORCHESTRATION (target)
   Phase 1 Acquisition: assemble the COMPLETE PRE fixture list for ALL mapped
     leagues.
   GATE: once the whole PRE set is complete/opened (pre-kickoff; NOT "all FT").
   Phase 2 Enrichment (batch, once): details, geo, team news, player usage,
     lineups, referee/discipline, etc.
   Phase 3 Value: compute value-panel value ONCE → FREEZE (never revised).
   DECIDED (user, 2026-07-01): the DETAILS assessment (3 above) MAY refresh closer
     to kickoff if needed (e.g. lineups / team-news land). The value-panel value
     (1) stays FROZEN once regardless.
